# iMAX POC Brief

## 한 줄 소개

iMAX는 증권사 현업의 질문을 업무지식 검색, 안전한 데이터 분석, 지출품의 처리로 연결하는 React + FastAPI 기반 통합 AI 에이전트입니다.

## 문제

현업에는 필요한 자료와 데이터가 이미 있지만, 매뉴얼 위치를 찾거나 SQL 작성자에게 조회를 요청하거나 품의 규칙을 다시 확인하는 과정이 반복됩니다. 단순 업무도 담당자 탐색, 요청, 대기, 수정, 재확인을 거치며 시간이 길어집니다.

## POC 범위

- LangGraph Supervisor와 세 전문 에이전트
- 매뉴얼 검색 결과만 사용하는 업무지식 답변과 파일·섹션 인용
- Semantic Layer, Schema Retrieval, LLM SQL 생성, SQL Validation을 거치는 NL2SQL
- role-based access control과 지점 범위 적용
- 데이터 설명, 핵심 수치, 자동 시각화, 가상 스크롤 표
- 지출품의 초안, 예산 검증, PDF 근거, 확인 토큰, 중복 실행 방지
- 세션과 LangGraph 체크포인트 영속화
- 라이트·다크 모드와 모바일·태블릿·데스크톱 반응형 화면
- 고정 시드 합성 데이터와 감사 로그

## 시연 질문

1. 회의실 예약 절차와 화면번호를 알려줘.
2. 지난 3개월간 지점별 신규 계좌 수 추이는?
3. 이번 달 고위험 상품 가입 건수가 많은 지점은?
4. 영업점별 ELS 가입 금액과 민원 건수를 비교해줘.
5. 스타벅스 88,000원 법인카드 품의해줘.
6. 전체 고객 원장과 계좌번호를 보여줘. 차단 시나리오

## 차별점

- 한 URL과 한 질문창에서 세 업무를 자연스럽게 이어갑니다.
- 모델 답변만 보여주지 않고 인용, 적용 기준, 차트, 표를 함께 제공합니다.
- SQL 실행과 지출 변경의 최종 결정은 각각 검증 계층과 결정론적 서비스가 담당합니다.
- 모호한 요청은 임의로 실행하지 않고 사용자가 업무를 선택하게 합니다.
- 기술 정보는 기본 화면에서 숨기고 필요할 때 분석 근거에서 확인하게 합니다.

## 안전 원칙

- 저장소와 시연에는 실제 고객, 계좌, 임직원, 지점 실적 데이터를 넣지 않습니다.
- 데이터 쿼리는 읽기 전용, 허용 스키마, 조회량, 역할·지점 정책을 통과해야 합니다.
- 지출 변경은 명시적인 확인 버튼과 유효한 확인 토큰이 있어야 합니다.
- 앱 상태, LangGraph 체크포인트, NL2SQL 조회 대상 DB를 분리합니다.
- `.env`, 실행 DB, 로그, 업로드 파일은 Git에서 제외합니다.

## 운영 확장

- 내부 승인 LLM 또는 내부망 LLM gateway
- 읽기 전용 DW/DM replica
- 사내 SSO와 trusted header 인증
- 부서·직무별 Semantic Layer와 권한 정책
- 사용자 피드백 기반 평가와 품질 회귀 체계
