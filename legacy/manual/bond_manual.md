## 2. 공시

**흐름**: 액션 → TR → 테이블 → 트리거 → 전송 → FEP → 수신 → 수신처리

- **TR4001 입력** → 트리거 → 공시=예정내역(4014) → `NSCG07M00` 생성 → `NPSC_SC3139U_INS`, `NPSC_SC3135U_DEAL_INS` 트리거 → `NPSC_SEND_DEAL_INFO`로 예탁원/협회 전송 → 거래소종목정보(`NSEA01M30`)/예탁원종목정보(`NSEA17M00`) 수신 → `NSCA02M00`(_IB/UB/DA) 갱신 → 상환일정 처리
- **매매(=공시=예정내역, 4014, 고객과 거래)**: 상품매수/매도(TR4215) → 동시결제시 계좌매도(1st 입금→매수, 2nd 출고) / 계좌매수(1st 입고→매도, 2nd 출금) → TR4215 & TR4047 대사(법인영업부, TR4047 처리)
  - 타사거래(자동처리): 인수/청약(=당발, 경과이자 X) 1차 16:30(배치~133), 2차 17:20(배치~134). 급할 땐 TR4029에서 결제 처리
  - 직매수/직매도: T+1결제(경과이자 O), 1차 익일 08:00, 2차 익일 10:30
- **단가수신 (4025 → NSCE04M00)**
  - clean단가(R시가단가 적용): TR4231 상품 말일자 평가시 사용, 수기처리(1차 dirty단가 insert 후 2차 clean단가 update)
  - 수신처: KIS/KAP/NICE/FNP (4군데)
  - dirty단가(EVLT_UV): 계좌평가시(하루 경과이자 포함) 사용
  - D일 기준 수신: EVLT_DT/EVLT_UV(더티단가, 경과이자 포함)는 익일자, TDY_EVLT_UV(클린단가, 경과이자 미포함)는 당일자
  - siga_gubun='1'(평균)은 별도 생성 필요, 나머지 구분자는 1주일 단위로 삭제

### 잔고
- 총: `NSCC01M00` (BD_FUND='00'). 수량은 천채 단위이나 잔고증명서 등에서는 함수로 원 단위 환산
  - BD_GDS_TP: 무조건 100으로 최초 생성. 200번대=REPO. 300~400은 안 씀. 500/600은 raw가 아닌 컬럼 update
- 펀드: `NSCF01M00` (BD_FUND='해당펀드')
- 히스토리: 2005-07-01 이전은 세후단가, 이후는 세전단가. 세후단가용이던 `NSCC01M90`은 현재 매도가능증권(=퇴직) 잔고로 활용. `NSCC01M00`은 단기매도증권(회계용어)
- Q) 원리금분배내역? — 공시/입출고 모두 보고서 때문에 TR4054에 입력해야 함
- 할인채: 잔고금액(=장부금액) = 매수금액 - 경과이자. 보유이자(= 미수이자+채권이자, 경과이자 미포함) 기준 과세. TR4183 총상품에서 월말마다 거래내역에 미수이자금액 생성(회계결산용)
- 매매금액 = 수량(천채) × 단가(만원) / 10 (함수 사용시 수량(원) × 단가(만원) / 10000)

### 트리거
- BD_DEAL_TP 값에 따라 E+1 처리 여부를 확인할 것

- 트리거: 공시(4014) → 잔고생성(TR4029)

### FI운용부 별도처리 (지준일 이슈)
- 4014 개별결제 → 4078
1. 예탁원/협회 둘 다 전송 안 함(NO)
2. 체결어제/결제오늘/시퀀스update는 어제날짜 대상으로 "어제 last seq + 1"
3. 현업: 원장정정/개별결제이므로 예탁원 NO, 협회는 공문 별도 전송 필요
- 지준일이 낀 경우에만 FI운용부만 예외 처리, TR4023 tab2 이용
- 다른 팀은 T일 4014에서 T+2(평시 T+1) 결제로 예탁원/협회 모두 YES 입력
- 예시(2/2가 지준일인 경우): 2/1에 TR4023 tab2로 미리 공시(다른 팀은 2/2 공시) → 2/2 협회NO/예탁원YES 공시 → 2/3 결제

### 거래최저단위
- TR4001의 거래최저단위가 1원이면 TR4014 단위도 원 단위로 통일 필요. 4001과 4014 단위는 반드시 일치(1원 or 1000원, default 1000원)

### 등록/공시 순서
- 3861에 먼저 등록 → 4014에서 계좌공시 (SF01OR0201: 3861 전송 OK 여부 확인)
- 종목등록 여부는 예탁원 관련. 당발물은 예탁원전송여부 'N'으로 우선 공시 → 4022에서 예탁원 미공시 처리 → 4023에서 협회 미공시 처리

### 협회미공시 (TR4023)
- 세전단가 소수점 무한루프 발생시: 우선 3자리까지 입력 후 4200에서 결제금액 처리. tab3/4에서 에러 확인(현업이 잘 확인 안 함)

### 별도
- 결제일/체결일 등 4014 수정시 RM 재처리 필요 (TR4057)

## 3. 종목정보

1. **4001**: `NSCA02M00` (당사)
2. **4111**: `NSEA01M30` (거래소, 무료) / `NSEA01M31` 거래소변경내역(TR4111 tab2)
3. **4114**: `NSEA17M00` (예탁원, 50만원)
   - 변동부이표체 정보를 제대로 안 줌 (참고: pjm ls *2022*)
   - 채권은 하루 3번(05시/12시/17시), CP는 1시간마다, 전단채는 10분마다 수신
   - CP/전단채는 15분 rule 없음

- 상장은 거래소 기준, 비상장은 예탁원 기준으로 처리
- 채권이 예탁원 미등록 & 당사 4001 미리 입력 후 매매 붙이는 경우(당발): 4014에서 예탁원 'N'으로 공시(하단 "직접" 표기 = 수기등록 의미) → 4022에서 나중에 예탁원 전송
- 장외: 예탁원 O, 협회 O, 거래소 X / 장내: 예탁원 O, 협회 X, 거래소 O
- 장내는 TR3851에 등록된 계좌로만 전송 (펀드:계좌 = 1:1 등록)
- 트리거: `NSCA02M00` → GEN_스케쥴
- 등록 종목이 많은 케이스(예: 기업금융부): TR4001에서 엑셀입력, 월 1회 정도 40~50종목

### 상환일정
- 이자일할계산기준: 해당없음 / 360 / 365(PI 때문에 생김) / 366. 상환일정은 3개월 단위로 생성
- CB(전환사채): 채권 전액매도 + 주식 수령 / EB(교환사채) / BW(신주인수권부사채)
- 상환일정 생성시 변동부이표 주의
- 4002에 등록되어 있어야 전환신청 가능 (예탁원에서 정보수신, TR4114 구분 '2')
- duration: 장외-사모채권은 의뢰 필요 (주관 현업 부서가 요청)
- CP/전단채/해외채권: 장내가 없으므로 4개 평가기관에 당사 잔고 송신 (국채는 제외, 전 증권사 공통 수신)
- 나중에 해외채권 USD 외 통화 공시시 해당국 세법 확인 필요 (정의 담당 부서 필요)
- 변동부 이표 소수점 6자리 초과시 GEN스케쥴 수정 필요 (현재 소수 6자리까지만 세팅 — 소스 파악 매우 중요). bd_chk에서 걸리면 상환일정에서 수기조정

### 오류정정 CASE
- T일: 예정내역 돌리고 신고
- T+1일: 매매불가기간(원칙상 매매불가, 예외적으로 풀어주고 매매 붙이는 경우 있음 → 결제에서 다시 신고)
- T+2일: 여기서도 잘못되면 재신고 불가 → 예탁원에 환급/추징 신청 (거래내역: 세금환급→세금추징 / 원천징수: 징수해지→환급→추징 — 환급 후 재추징 방법 파악 필요)
- 문서로 오는 예외 케이스: 지점 입금 이후 재무부에서 수정처리

### 위임여부 선택
- 재무부 문서처리 간소화 대비: `NCZA01M00`(tax_trst_tp) 위임여부 체크 → 체크된 건은 원천징수 생략 (근거문서는 지점 보관 필요). 위탁만 대상, 상품분은 무조건 원천징수
- cf) 4067에서 CD는 채권과 동일 처리, CP/전단채는 위임여부 필드 이미 존재

## 4. 계좌거래

- 4019에서 위탁공시 (4014의 위탁 버전과 동일). 차이점: 4014는 공매공시도 가능
- 증거금징수계좌는 오직 4019에서만 처리 (약속)

### 공시 종류
| 종류 | 조건 |
|---|---|
| 일반공시 | 상품:계좌 = 1:N, TR4018, `NSCG01M00` |
| 합의공시 | 상품:계좌 = 1:1, TR4019, `NSCG07M00` |

- 일반공시 계좌구분: 6X(세금우대 폐지됨), 6X 저축(일반)=저축계좌만, 6X 저축(비과세)=비과세계좌 대상, 11/91=일반(ISA 포함)
- TXPF_YN: N=일반, Y=세금우대(폐지), S=생계형(=비과세)
- CD/CP/전단채는 `NCKE06M00` 사용 — cust_seq 있으면 합의공시, 없으면 일반공시
- 랩: 계좌매수는 일반/합의 모두 가능, 계좌매도는 합의로만. 전체적으로 계좌매도·상품매수는 합의로만 진행(일반은 사고 가능성 있음 — 사전 차단 검토 필요)

### 계좌 공시+매매 동시처리
- TR4010 (`CK401002`) → `SC3128U_CO_BUY`
- 장외매매는 cyber_tp='1'(온라인) 매매 불가. 일반공시일 때 예약 체크
- 매매불가기간(전화문의 많음): TR4000에서 term 정의. 장내는 항의가 많아 체크 제외
- `NPCK_CK3128U_BUY` 처리: ① 예수금체크 → ② 상품거래내역(`NSCF02M00`, `NSCC02M00`) → ③ 계좌별 거래내역 `NSCC03M00` → (트리거) `NPSC_GEN_TRDE` 호출 → `NPCA_CAC05M00` → `NCAC05M00`
  - Q) 나중에 RP 관련 가능금액 계산시 무한루프 이슈 있음 — 반드시 확인 필요
- 합의공시 `NSCG07M00` / 일반공시 `NSCG01M00` — 둘 중 한 곳에만 저장
- `~IO`, `~SUDO` 프로시저(npca)는 위탁과 채권이 공유 — 수정시 매우 주의

### 랩 예시
1. 4014 → 4019(Q2)펀드 계좌별로 5건씩 입력 → 6652 하단에 표시 → 실제 6652 매수시 4019 입력된 합의계좌 체크
2. 4019 합의공시(펀드·계좌) → 4010 계좌매수 진행 → 거래내역 생성

| | 거래 조회 | 잔고 조회 |
|---|---|---|
| 상품 | 4295 | 4231 |
| 위탁 | 3356 | 4122 |

- 4019 상품매수공시(tab2): 매수수익률 = 할인율(높을수록 싸게 매수, 차액은 상품전략부 수익. 4287에서 지점판매보수 지급)
- 4010 계좌매도 진행: 상품선택시 세금계산 `NPSC_BD_INT_TAX` → `NPSC_CALC_INT_TAX` → `~POSS_INT`(`NPSC_POSS_BD_INT`)가 메인 로직. 입력은 `3128~SELL_PROC`
- 4010 실제 매도 전 세금 예측용 가상매매: TR4189

### 소액부징수

- 개인: 소액부징수 없음. 단, 10원 미만은 세금 안 뗌

## 7. 자동/반자동 처리

- 처리 순서: 거래내역 → 잔고 → 종목정보 → 상환일정
- 독학 필요 영역: 4014에서 수익률 입력시 세전단가 계산 로직 (나중에 세후단가로 바뀌면 총이자에 경과이자 추가 — 현재는 00. 세후단가는 66=총이자, 99=보유이자)
- 자동 vs 반자동은 회계적 이슈. 체결은 무관, 결제만 상환 여부 관련
- 자금팀이 TR5411(자금판)을 빨리 보기 위해 반자동을 만듦. 직거래만 반자동 있고 계좌는 마감시 처리
- TR5220에서 직거래는 별도로 빼서 마감회계처리. `NEAG06M00` 선행. cf) TR5225는 자동
  1. `NSCC02M00`(단기) — `NPEA_GDS_SC_STAT`(TR5220 직거래전표 반자동 생성)
  2. `NSCC01M90`(매도가능) — `NSCF02M00` (매도가능만 기타포괄손익 사용) / 차입

### D+1일 수정 절차
1. 4029 취소 (이전 회계전표 취소 먼저)
2. 4001 단위당 이자액 수정 (경과이자 재계산)
3. 4029 재결제
- 수정 순서 전체: ① 결제취소 TR4200 → ② tab1에서 수정 → ③ 협회/예탁원 수정 요청(현업/결제업무부, 전문이 자동으로 안 나가므로 별도 통보 필요) → ④ RM 재처리 → ⑤ 5818 재처리
- Tip: 수정시 `NSCC~` 원장정정내역 확인하며 `NEAG06M00`도 같이 update 확인
- 지준일(TR4031)에는 결제 불가. 현업은 D+1일 취소/수정 불가

### 결제취소
| 시점 | 절차 |
|---|---|
| 배치 돌기 전 | ① 공시삭제 → ② RM재처리(4057, 체결잔고 재생성) → ③ 5818 `~PSIT_INFO_SC` 재처리(사전에 경열 문의) |

### 1주일 전 직거래 단위당 이자 수정 (최악의 케이스)
1. 자금팀 5220 전표번호 취소
2. 해당거래 update (총거래/펀드거래/`NEAG06M00` 3개)
3. 회계IF에 직거래전표 재처리 요청

- 2.1~2.10: 타인분경과이자 / 2.10~2.20: 보유이자(=미수이자+채권이자) / 2.1~2.20: 경과이자(발행~오늘)

## 8. Invalid (비정상거래)

- 취소거래 처리시 `NSCC02M10`↔`NSCC02M00`, `NSCF02M10`↔`NSCF02M00` 각각 한 쌍
- 문제: 취소된 거래(3번째)가 있고 그 다음 거래(4번째)가 있으면, 4번째 거래부터 경과이자/순수장부금액이 잘못 계산됨

### 처리 프로세스
1. 취소된 건의 stat_tp를 '1'로 update
2. 마감할 때 걸림
3. TR4016에서 비정상거래 처리 (`NPSC_PROC_INVALID`) — 결제 또는 IT 처리

### TR4032
- 현재 미사용. FI운용부에서 상품전략부로 이관. 순서는 4030-4032(현재 사용 X)

### 경과이자 발생 후 취소해야 하는 경우
1. 예: 4.00%가 맞는데 결제일에 예탁원에서 4.05%(오류)로 전송
   - 8시 43분에 예탁원 종목정보 수신
   - 다시 생성해야 하는 거래의 이전 거래를 update (`NSCF02M00` stat_tp)
   - `CK410704` 처리 후 로그 반드시 확인
2. TR4016 처리 전: 4001 이자율 정확히 세팅 후 생성 버튼으로 처리
3. TR4295에서 확인: 재조회 전에 출력물 미리 확보(뽑히지 않을 경우 대비) 후 재조회하여 대사
